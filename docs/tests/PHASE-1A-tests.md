# PHASE 1A — Tests: Project Scaffolding

**Phase goal:** WXT + TypeScript + React + Tailwind toolchain compiles, builds, and produces a loadable Chrome MV3 extension skeleton.

---

## Manual Test Checklist

Run from the repo root: `/Users/claude/repos/slingshot`

### Toolchain
- [ ] `node --version` reports v18 or higher
- [ ] `npm --version` reports v9 or higher

### Install & generate types
- [ ] `npm install` completes without errors (warnings are OK)
- [ ] `node_modules/wxt/` exists
- [ ] `.wxt/tsconfig.json` exists (generated by `postinstall: wxt prepare`)
- [ ] `.wxt/types/imports.d.ts` exists and declares `defineBackground`

### Type checking
- [ ] `npx tsc --noEmit` exits with code 0 and no output
- [ ] No `any` types are used in `src/entrypoints/background.ts`

### Production build
- [ ] `npm run build` succeeds
- [ ] `build/chrome-mv3/manifest.json` exists
- [ ] `build/chrome-mv3/background.js` exists
- [ ] Manifest contains `"manifest_version": 3`
- [ ] Manifest contains permissions: `contextMenus`, `storage`, `activeTab`, `clipboardWrite`
- [ ] Manifest contains host permission `<all_urls>`

### Dev mode
- [ ] `npm run dev` starts a Vite dev server on `http://localhost:3000`
- [ ] Dev mode auto-opens a Chrome window with the extension loaded
- [ ] Console output shows `Built extension in <NN> ms`
- [ ] `Ctrl+C` stops the dev server cleanly

### Load in Chrome (the real acceptance test)
- [ ] Open `chrome://extensions/`
- [ ] Enable **Developer mode** (top-right toggle)
- [ ] Click **Load unpacked**
- [ ] Select `/Users/claude/repos/slingshot/build/chrome-mv3/`
- [ ] **SLINGSHOT** appears in the extensions list with no errors
- [ ] No red **Errors** badge on the extension card
- [ ] Click **Inspect views: service worker** — DevTools opens
- [ ] Console in service worker DevTools shows `[SLINGSHOT] background service worker booted (Phase 1A skeleton)`
- [ ] Extension card shows the four declared permissions when expanded

### Repository hygiene
- [ ] `.gitignore` excludes `node_modules/`, `.wxt/`, `build/`
- [ ] `git status` shows only the 5 config files + `.gitignore` + `src/entrypoints/background.ts` + `src/entrypoints/.gitkeep` as untracked / modified

## Notes

- The 5-file Phase 1A deliverable list (`package.json`, `wxt.config.ts`, `tsconfig.json`, `tailwind.config.js`, `postcss.config.js`) was extended by **one** file, `src/entrypoints/background.ts`, because WXT refuses to build with zero entrypoints. - `.gitignore` was added as part of repo hygiene; it is not counted as a Phase 1A deliverable but is necessary so that `node_modules/` and build outputs do not pollute the next commit.
